How to Use

What you want to do is drag the "TextMeshAnimator.cs" script onto the gameObject that contains the "Text Mesh Pro UGUI (Script)" component.


After inserting the component, you should see this in the gameObject's inspector:

TMP Text or Custom Text

In TextMeshPro, you can assign text to the main inspector panel. However, if you want to provide custom text in the TMP Animator script, you may do so by checking the box "Custom Text" and s text area control will appear in the panel.

Custom text will be parsed and assigned to the actual text of TextMeshPro component when the game starts. This is useful for when you want to visualize what the final text will look like. One example is whether you're in game mode or edit mode, you can type text in the "Custom Text" text area, press update, and it will show you the parsed result.

TextMeshPro Animator does not support '<' or '>' displays because they are what are used for defining tags.

Tags

As you may have noticed in the last visual, I only used one tag. Unlike xml, these tags do not require a following closing tag such as "<tag></tag>". Tags also support multiple commands within them.
<effectType effectType2 modifier=5.0 Independency>My Text

Effects

Shake

The shake effect can be applied to the text by adding the <shake> tag right before the text you wish to shake.

I am <shake>SHAKING!!!

You can terminate the shake effect by adding the </shake> tag right after the text you wish to discontinue shaking.

I am <shake>SHAKING</shake> somewhat?

You can take advantage of where to place these opening and closing tags.

This is o<shake>d</shake>dly st<shake>range...

Now let's talk about the modifiers of this shake effect. The shake effect supports two different modifiers:

  • Intensity

    Defines how massive the shake is. The modifier is written as "Intensity=0" and is not case sensitive. Negative values are not allowed. Invalid value characters result in a fine of $50 minimum... Psst HA HA I'm joking of course. Invalid value characters result in an error. This example should show you how to use it in a tag:

    I <shake Intensity=0.25>feel <shake INTENSITY=0.1>so <shake intensity=1>weird...

  • Independency

    Defines whether the text will shake by vertices, characters, words, or if the whole will shake as one whole. The modifier is literally written as either "Vertex", "Character", "Word", or "United" and is not case sensitive. This example should show you how to use it in a tag:

    <shake WORD>Bippity <shake CHARACTER>Boppity <shake VERTEX>Boo

You can also assign modifiers inspectorly. The inspector values will be multiplied by the values specified in the tags. If the values are not specified by tags, they will use the inspector values directly.
The shake Independency value requires the text to be updated. You can update it by clicking "Update".

Wave

The wave effect can be applied to the text by adding the <wave> tag right before the text you wish to wave up and down.

<wave>God Bless America

You can terminate the wave effect by adding the </wave> tag right after the text you wish to discontinue waving.

King Boo says "<wave>OOOH</wave>" to haunt Luigi.

You can take advantage of where to place these opening and closing tags.

This is o<wave>d</wave>dly st<wave>range...

Now let's talk about the modifiers of this wave effect. The wave effect supports four different modifiers:

  • Intensity

    Defines how high the text waves up and down. The modifier is written as "Intensity=0" and is not case sensitive. Negative values are not allowed. Invalid value characters result in an error. This example should show you how to use it in a tag:

    I <wave Intensity=0.5>feel <wave INTENSITY=0.1>so <wave intensity=1>weird...

  • Speed

    Defines how fast the text waves up and down. The modifier is written as "Speed=0" and is not case sensitive. Negative values are not allowed. Invalid value characters result in an error. This example should show you how to use it in a tag:

    <wave speed=0.25>I am <wave speed=1>speed!!!

  • Separation

    Defines how far apart each independency delays eachother, resulting in how wide the wave looks. The modifier is written as "Separation=0" and is not case sensitive. Negative values are not allowed. Invalid value characters result in an error. This example should show you how to use it in a tag:

    Sometimes, <wave separation=1>I can get... <wave separation=0.25>FUNY! <wave speed=4 separation=0.1>:D

  • Independency

    Defines whether the text will wave by vertices, characters, words, or if the whole will move up and down as one stiff whole. The modifier is literally written as either "Vertex", "Character", "Word", or "United" and is not case sensitive. This example should show you how to use it in a tag:

    <wave WORD>Bippity <wave CHARACTER>Boppity <wave VERTEX>Boo

You can also assign modifiers inspectorly. The inspector values will be multiplied by the values specified in the tags. If the values are not specified by tags, they will use the inspector values directly.
The wave Independency value requires the text to be updated. You can update it by clicking "Update".

Wiggle

The wiggle effect can be applied to the text by adding the <wiggle> tag right before the text you wish to wiggle. You may think that this is like the shake effect. The wiggle effect is a little more advanced and smooth. Not gonna lie though, they are very similar and the wiggle effect can substitute for the shake effect.

<wiggle>The Wiggles

You can terminate the wiggle effect by adding the </wiggle> tag right after the text you wish to discontinue wiggling.

<wiggle>Wiggle wiggle wiggle wiggle wiggle!</wiggle> Yeah!

You can take advantage of where to place these opening and closing tags.

This is o<wiggle>d</wiggle>dly st<wiggle>range...

Now let's talk about the modifiers of this wiggle effect. The wiggle effect supports four different modifiers:

  • Intensity

    Defines how far out the text wiggles around. The modifier is written as "Intensity=0" and is not case sensitive. Negative values are not allowed. As normal, invalid value characters cause an error. This example should show you how to use it in a tag:

    I <wiggle Intensity=0.5>feel <wiggle INTENSITY=0.1>so <wiggle intensity=1>weird...

  • Speed

    Defines how fast the text wiggles. The modifier is written as "Speed=0" and is not case sensitive. Negative values are not allowed. Invalid value characters result in an error. This example should show you how to use it in a tag:

    <wiggle speed=0.25>I wiggle. :) <wiggle speed=1>I wiggle faster!!! :D

  • Minimum Duration

    Defines how smooth the text wiggles. The modifier is not written in tags but rather instead defined directly from the inspector variable. The higher the value, the steadier the wiggle, but the more likely to make the wiggle seem more machinacial. The lower the value, the glitchier the wiggle looks, but brings more dynamic to the effect.     <wiggle WORD>Bippity <wiggle CHARACTER>Boppity <wiggle VERTEX>Boo

You can also assign modifiers inspectorly. The inspector values will be multiplied by the values specified in the tags. If the values are not specified by tags, they will use the inspector values directly.
The wiggle independency value requires the text to be updated. You can update it by clicking "Update".

Scripting

Simplest way that I can say this is that the function TextMeshAnimator.BeginAnimation(text) is what I suggest the most to call when you trigger a dialog box, and if I ever get to opening, closing transitions in the future, there will be a TextMeshAnimator.EndAnimation() available to close the dialog.
